# Storefront Password Auto-fill - Demo Script

## Demo Setup

### Prerequisites

1. A password-protected Shopify storefront (URL ending in `/password`)
2. Alfred extension installed in Chrome/Edge
3. Clear any existing saved passwords for the demo store

### Recommended Recording Settings

- Screen resolution: 1920x1080 or 1280x720
- Focus on browser window only
- Disable desktop notifications
- Use cursor highlighting (optional)

---

## Demo Script

### Part 1: The Problem (10 seconds)

**Visual:** Navigate to a password-protected Shopify store

**Script:** "When working with Shopify development password-protected stores,
you constantly need to enter passwords. This gets tedious fast, especially when
working with multiple stores."

---

### Part 2: Saving a Password (15 seconds)

**Visual:** Open Alfred popup, show storefront password section

**Actions:**

1. Click Alfred extension icon
2. Scroll to "Storefront password" section
3. Enter password in the field
4. Check "Auto-fill password?" checkbox
5. Click "Save" button

**Script:** "With Alfred, just open the extension popup, enter your storefront
password, enable auto-fill, and save."

---

### Part 3: Auto-fill in Action (10 seconds)

**Visual:** Reload the password page or navigate to it fresh

**Actions:**

1. Navigate to the `/password` page
2. Show password being auto-filled instantly
3. Form submits automatically

**Script:** "Now when you visit the password page, Alfred automatically fills
and submits the password. No more manual entry."

---

### Part 4: Management (10 seconds)

**Visual:** Open popup again, show toggle and edit options

**Actions:**

1. Show disabling auto-fill with checkbox
2. Show editing password
3. Show deleting password (clear field + save)

**Script:** "Easily manage passwords from the popup: toggle auto-fill, update
passwords, or delete them entirely."

---

## Total Duration: 45 seconds

---

## Key Points to Emphasize

1. **Local Storage** - Passwords stored locally in your browser, not on any
   server
2. **Domain-Specific** - Each store gets its own password entry
3. **Automatic** - Zero-click experience once configured
4. **Smart Failure Handling** - Auto-disables if password fails (prevents loops)

---

## Optional: Extended Demo (Settings Page)

### Part 5: Settings Management (10 seconds)

**Visual:** Open Alfred settings page

**Actions:**

1. Click "Settings" in popup
2. Show "Storefront Passwords" section
3. Display table with all saved passwords
4. Show edit/delete/enable/disable actions

**Script:** "Manage all your storefront passwords from the settings page with a

## Recording Tips

1. **Clean browser** - Use incognito mode or clean profile
2. **Slow down** - Pause briefly after each action
3. **Mouse movements** - Smooth, deliberate movements
4. **Audio** - Record voice-over separately for clarity
5. **Transitions** - Use subtle fade between sections
6. **End card** - Show Alfred logo or website
